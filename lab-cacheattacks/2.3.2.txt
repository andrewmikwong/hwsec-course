**Discussion Question 6:** Describe your communication protocol. Please refer to Section 2.2 for the components involved in a protocol. 

So for the 8-bit integer across processes, we used a shared L2 cache as the communication medium and we encoded one 8-bit integer using the set-granularity prime-evict-probe. The sender and receiver both allocate a 2 MB page so that relevant address bits are under control, and both construct linked lists of addresses that map to specific L2 sets. We reserve sets 0-7 for the data bits and set 8 as a valid/start signal. 

For encoding, the sender will take an integer from [0, 255] and interpret as 8 bits. During the transmission window, the sender will repeatedly access addresses mapping to set 8 and this indicates that a message is present. For each data bit with a 1 value, the sender will then repeatedly access the addresses for that corresponding set among sets 0-7. For each data bit that is 0, the sender leaves that set idle. In this way, heavy activity on a set represents bit 1, while no activity represents bit 0. 

For decoding, each set will use a threshold obtainted during callibration from repeated no-contention measurements. If a set's measured latecny is above its threshold, the receiver will interpret the sample as evidence that the sender accessed the set. Over the course of many samples, the receiver will then use majority voting which if a data set is above threshold in most valid samples, that bit is decoded as 1, otherwise decoded as a 0. The reciever will then reconstruct the integer from the 8 decoded bits, then waiting until the valid bit on set 8 drops for several consecutive checks before listening for the next message
